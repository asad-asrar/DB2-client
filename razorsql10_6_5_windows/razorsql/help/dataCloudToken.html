Salesforce Data Cloud Authentication

There are two methods of authentication supported for Salesforce Data Cloud. They are the following: 1. Authentication with a User name, Password, Client ID (Consumer Key), and Client Secret (Consumer Secret)

2. Authentication with an OAuth token, refresh token, Client ID (Consumer Key) and Client Secret (Consumer Secret)

If selecting the Data Cloud Access Token authentication option, the following information will need to be entered: An OAuth token, a refresh token, and the Client ID and Client Secret from the Salesforce connected app.

For information about generating OAuth tokens, please see the following URL:

https://github.com/forcedotcom/Salesforce-CDP-jdbc/blob/develop/README.md

If selecting the Data Cloud User and Password authentication option, enter the user name (generally an email address) for Salesforce Data Cloud database and the password corresponding to the user name. You will also need to enter the Client ID and Client Secret values from the connected app in Salesforce.
